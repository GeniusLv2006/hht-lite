# Documentation

The root directory keeps the files that GitHub and common open-source tooling expect to find there: `README.md`, `CONTRIBUTING.md`, `SECURITY.md`, `LICENSE`, and `THIRD_PARTY_NOTICES.md`.

Additional project documentation is organized here:

- [Maintenance guide](maintenance.md): routine verification, version synchronization, deployment, and rollback.
- [Release process](releasing.md): version policy, review boundaries, deployment, tags, and GitHub Releases.
- [Source provenance audit](provenance.md): the v6 independent implementation baseline and its audit method.

Self-hosting, backup, upgrade, and troubleshooting instructions remain in the root [README](../README.md) because they are part of the primary onboarding path.
